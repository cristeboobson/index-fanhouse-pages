# index-fanhouse-pages

一个用于归档和发布多个独立 HTML 页面的仓库。  
本项目不针对任何特定域名或网站，仅作为静态页面的集合与索引。

## 项目简介

本仓库存放了若干独立的 HTML 页面，这些页面各自独立、功能各异，不构成一个统一的网站或应用。  
仓库的主要目的是：

- 归档历史页面副本
- 提供可直接访问的静态页面入口
- 方便页面内容的版本管理与回溯

## 目录说明

```
.
├── index.html          # 页面索引（如有）
├── pages/              # 存放独立 HTML 页面的目录
│   ├── page-001.html
│   ├── page-002.html
│   └── ...
├── assets/             # 共用资源文件（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
├── README.md           # 本文件
└── LICENSE             # 许可证文件
```

- `pages/`：每个 HTML 文件为一个独立页面，彼此之间无依赖关系。
- `assets/`：存放页面可能引用的公共资源，部分页面也可能自带内联资源。
- `index.html`：如存在，则为页面列表或导航入口。

## 页面归档说明

- 每个页面均保留原始结构，不做功能修改或内容增删。
- 归档时间戳、页面来源等元信息（如有）会记录在页面注释或独立元数据文件中。
- 部分页面可能因外部资源失效而无法完整渲染，此为归档过程中的常见情况。

## 维护说明

- 本仓库不定期更新，新增页面会按时间或主题归类放入 `pages/` 目录。
- 欢迎通过 Issue 提交页面归档请求，但需确保页面内容不违反相关法律法规。
- 不接受对已有页面的功能性修改请求，仅接受链接修复或明显错误更正。
